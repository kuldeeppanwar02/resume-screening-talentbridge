Problem Statement
Why TalentBridge needs
an automated solution

Inconsistency

Feature Engineering
15+ engineered signals for precise scoring
Skill Features
mandatory_match_pct

% of must-have JD skills present in resume (weighted 70%)

optional_match_pct

% of good-to-have skills matched (weighted 30%)

skill_score

Final: 70% mandatory + 30% optional → 0–100

Experience Features
experience_score

Ratio of candidate exp vs JD minimum, capped at 100. Mean dataset: 6.73yr

experience_bucket

Junior · Mid · Senior · Lead (45.2% are Senior)

seniority_align

Binary: does bucket match JD seniority requirement?

Context Features
domain_match

Candidate's domain = target domain? (6 domains in dataset)

edu_total

Education level + institution tier bonus (Tier-1: +10, Tier-2: +5)

ats_composite

Average of all 16 binary ATS flags (e.g., Agile 78.3%, Cloud 27.8%)

Rule Score Formula

Rule_Score = (skill_score × 0.40) + (experience_score × 0.25) + (domain_match × 0.15) + (edu_total × 0.10) + (ats_composite × 0.10)

NLP Pipeline
TF-IDF semantic similarity scoring
Processing Steps
1
Text Extraction

Combine tech_skills + experience_summary + project_summary + key_achievements into one resume document.

2
NLP Preprocessing (NLTK)

Lowercase → remove special chars → tokenize → stopword removal → WordNet lemmatization.

3
TF-IDF Vectorization

1,000 features · bigrams (1,2) · min_df=2. Converts resume and JD text to comparable numerical vectors.

4
Cosine Similarity

Score each resume vector against the JD vector. Normalized 0–100. Contributes 40% of final hybrid score.
